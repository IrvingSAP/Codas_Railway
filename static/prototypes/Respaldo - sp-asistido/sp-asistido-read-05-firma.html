5 · Identificación y contrato (IBM i · §9.7)

Indique esquema, nombre corto y nombre largo del procedimiento; son obligatorios. La lista de filtros (paso 3) y SPECIFIC / plantilla de salida se resumen abajo. En el producto se persiste la sesión del asistente.

Procedimiento almacenado (obligatorios: esquema, nombre corto, largo)

Atributo: name="procedure_schema"

Atributo: name="procedure_name_short"

Atributo: name="procedure_name_long" — no puede quedar en blanco.

Comentario (opcional) — no bloquea el paso; implementar en producto si aplica

— (en ADD existe procedure_comment)

SPECIFIC (objeto en catálogo · §9.3)

SELCUS0SP1

Result set (§9.7)

El resultado tabular se devuelve vía un cursor de salida (DYNAMIC RESULT SETS 1). No se transportan filas en P_ERROR_MSG.

Parámetros IN (derivados del paso 3: siempre al menos un criterio WHERE — un P_* o literal según origen)

Ej. P_ESTADO CHAR(1) si el filtro es sobre CUSTATUS con parámetro IN

Parámetros OUT (plantilla · §9.2)

P_ERROR_CODE INTEGER, P_ERROR_MSG VARCHAR(200)

Incluir en el script (plantilla §9 + lectura)

  • DYNAMIC RESULT SETS 1 y cursor con FOR SELECT …
  • OUT P_ERROR_CODE y OUT P_ERROR_MSG
  • Manejador 23505 (si aplica) y SQLEXCEPTION (general)
