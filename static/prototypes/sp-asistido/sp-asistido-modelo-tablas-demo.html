SP Asistido · Tablas propuestas (sin programación)

Diseño base para persistir definiciones, pasos, parámetros, criterios y versiones SQL de ADD/READ/UPD/DLT.

Relaciones clave

Company 1..N SPDefinition · HeaderTable 1..N SPDefinition · SPDefinition 1..N (StepState, Parameter, Assignment, Condition, ArtifactVersion)

Auditoría y emisión de script (alineado a HeaderTable)

  • Todas las tablas propuestas incluyen created_at, updated_at, created_by, updated_by (ForeignKey a usuario, on_delete=SET_NULL, null=True).
  • En cabecera (sp_asistido_spdefinition): script_generated y script_date indican si ya se emitió/confirmó el script del SP (misma idea que DDL en table design).
